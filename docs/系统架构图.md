## 二、初始化时序

```mermaid
sequenceDiagram
    participant U as 用户
    participant FE as 配置页
    participant API as Next.js API
    participant SDK as 飞书 Node SDK
    participant DB as Postgres

    U->>FE: 1. 点击创建应用
    FE->>API: POST create-app
    API->>SDK: registerApp(addons)
    SDK-->>FE: 创建链接 / 二维码
    U->>SDK: 在飞书确认创建
    SDK-->>API: App ID / App Secret
    API->>DB: 立即加密落库
    API->>SDK: 配置回调、WS、Refresh Token 并发布

    U->>FE: 2. 点击用户授权
    FE->>API: POST authorize/start
    API->>DB: 保存一次性 OAuth state 哈希
    API-->>FE: authorizationUrl
    U->>SDK: 同意授权
    SDK->>API: GET oauth/callback?code&state
    API->>SDK: 交换并读取用户令牌/身份
    API->>DB: 加密保存 Access/Refresh Token
    API-->>FE: 重定向配置页

    U->>FE: 3. 选择组织
    FE->>API: PATCH selectedOrgTargetId
    API->>DB: Base/权限/事件状态置 pending
    API->>SDK: 4. 用户身份读取目标 Base
    API->>SDK: 5. 前四项通过后建立 WSClient
    SDK-->>API: onReady
    API->>DB: 标记初始化完成
    FE-->>U: 事件状态跃迁成功，展示烟花
```

## 三、隔离与恢复

- 每个业务调用显式传递 `integrationId` 或 `integrationContext`。
- App Secret 与 OAuth Token 按 integration 加密落库，不依赖容器目录。
- 服务启动时只为数据库五段门禁满足的 integration 恢复 WebSocket。
- 事件任务、重试和恢复以 `meeting_pipeline_tasks` 为事实来源；WebSocket Map 只是运行态缓存。
- Base 访问和报告读取都绑定当前 integration 与组织目标。
