## 核心链路

1. 手动入口：用户上传会议纪要，调用 `src/app/api/analyze/route.ts`。
2. 自动入口：飞书会议事件通过 `src/app/api/feishu/webhook/route.ts` 进入系统。
3. `analysisService.ts` 统一调用豆包 API，返回结构化分析结果。
4. `webhookProcessor.ts` 拉取逐字稿、调用分析服务，并回写多维表格。
5. `src/instrumentation.ts` 在服务启动时触发飞书链路恢复扫描。
6. `src/lib/feishu/monitor.ts` 统一输出飞书搜索、导出、分析和恢复的结构化日志。
7. 报告页 `src/app/report/page.tsx` 通过 `recordId` 从 Base 读取 `JSON数据` 渲染报告。
