## 核心文件说明

### API 路由

| 文件 | 用途 |
|------|------|
| `src/app/api/analyze/route.ts` | 会议分析接口：文件解析（.txt/.docx）→ LLM 调用 → 后端验证（决策树 Zone 判定 + LLM 重写兜底）→ 返回结果 |
| `src/app/api/feishu/webhook/route.ts` | 飞书 Webhook 接收入口：URL 验证、token 校验、事件去重、异步入队 |
| `src/app/api/feishu/record/route.ts` | 使用飞书 OpenAPI 根据多维表格 recordId 读取报告 JSON，用于 `/report?recordId=xxx` 渲染 |

### 组件

| 组件 | 用途 |
|------|------|
| `AnalysisDashboard` | 分析结果展示仪表盘，包含图表、行为分析、给领导者的建议、姓名隐藏、PDF 导出 |
| `InputForm` | 文件上传表单，支持 .txt/.docx，先添加到待分析列表，点击"开始分析"后提交 |
| `src/app/report/page.tsx` | 统一飞书报告入口，从多维表格 `JSON数据` 字段读取 AnalysisResult 并复用 AnalysisDashboard |
| `Layout` | 应用布局组件 |
| `BehaviorRadar` | 四维行为雷达图（直言不讳、协同、实验、反思），低分兜底 + 数据点标记 |
| `CommunicationBubbleChart` | 沟通气泡图，展示发言分布 |
| `TeamStateChart` | 团队状态四象限图 |

### 常量配置

| 文件 | 用途 |
|------|------|
| `teamingRules.ts` | 组队分析规则，包含完整的 LLM 系统提示词（含决策树规则、行为评估、analysis 书写指引） |
| `zoneConfig.ts` | 决策树 Zone 判定函数（determineZoneFromBehaviors）+ analysis 重写（LLM 重写优先，硬模板兜底） |
| `behaviorConfig.ts` | 行为级别配置（Red/Blue/Green/Grey 颜色与图标）+ 行为描述定义 |

### 类型定义

| 类型 | 用途 |
|------|------|
| `AnalysisResult` | 分析结果完整数据结构 |
| `TeamBehaviors` | 四维行为评估（speakingUp/collaboration/experimentation/reflection） |
| `TeamState` | 团队状态（Zone + PS/WS 分数 + analysis） |
| `LeaderAdvice` | 给领导者的建议（含行动类型 action、建议内容 advice、综合论述 reasoning） |
| `BatchItem` | 批量分析项（文件 + 状态 + 结果） |
