## 根目录配置文件

| 文件 | 用途 | 说明 |
|------|------|------|
| `package.json` | 依赖管理 | 定义项目依赖和脚本；生产构建固定使用 Webpack 兼容飞书 SDK 与现有 Babel 配置 |
| `pnpm-lock.yaml` | 依赖锁定 | 锁定依赖版本 |
| `tsconfig.json` | TypeScript 配置 | TypeScript 编译器配置 |
| `next.config.ts` | Next.js 配置 | Next.js 运行配置 |
| `eslint.config.mjs` | ESLint 配置 | 代码规范检查配置 |
| `Dockerfile` | 容器构建 | 固化 Node/pnpm/Next.js 运行环境，便于迁移 |
| `docker-compose.yml` | 容器部署 | 将宿主机 `127.0.0.1:3011` 映射到容器 `3000` |
| `.env.production.example` | 环境变量样例 | 平台级配置、豆包 API、公网域名与任务恢复开关 |
| `drizzle.config.ts` | 数据库配置 | Drizzle 针对 Supabase Postgres 的 schema/generate/push 配置 |

## 核心链路

1. 手动入口：用户上传会议纪要，调用 `src/app/api/analyze/tasks/route.ts` 创建 `web_analysis_tasks` 异步任务，前端轮询 `src/app/api/analyze/tasks/[taskId]/route.ts` 获取结果。
2. 配置入口：`src/app/feishu-config/page.tsx` + `src/components/FeishuConfigWorkspace.tsx` 展示飞书接入五步向导；交互只更新局部状态，不执行整页刷新。
3. 用户级配置入口：`src/app/api/feishu/integrations/*` 负责按登录用户读写飞书集成配置。
4. 应用创建入口：`src/app/api/feishu/integrations/create-app/route.ts` 通过 SDK `registerApp` 创建用户独立应用，结果立即加密落库，并自动配置/发布应用。
5. 用户授权入口：`src/app/api/feishu/integrations/[integrationId]/authorize/start/route.ts` 生成标准 OAuth 地址；`src/app/api/feishu/oauth/callback/route.ts` 换取并加密保存真实 Token。
6. 初始化编排：应用校验、用户 Token、组织、Base 可访问和用户权限依次通过后，才允许启动事件长连接；所有门禁状态写入数据库。
7. 事件校验入口：`src/app/api/feishu/integrations/[integrationId]/event-subscription/check/route.ts` 启动 SDK `WSClient` 并等待 ready，ready 是完成烟花的唯一触发状态。
8. `analysisService.ts` 统一调用豆包 API，返回结构化分析结果。
9. `webAnalysisTaskStore.ts` / `webAnalysisTaskProcessor.ts` 负责网页上传任务的落库、领取、执行、完成/失败状态更新。
10. `eventListenerManager.ts` 负责按集成建立 SDK 长连接、消费妙记生成事件，并把事件写入数据库任务表；未满足前置门禁的集成不会启动。
11. `meetingPipelineTaskStore.ts` 负责承接会议任务的落库、领取、重试计划、完成/失败状态与启动恢复读取。
12. `meetingPipelineWorker.ts` 作为常驻任务执行器，轮询数据库领取到期任务，再执行 `minute_token -> transcript -> analysis -> Base 回写` 主链路。
13. `src/instrumentation.ts` 在服务启动时先拉起任务执行器，再基于数据库任务表执行启动恢复。
14. `src/lib/feishu/monitor.ts` 统一输出飞书录制文件、导出、分析和恢复的结构化日志。
15. 报告页 `src/app/report/page.tsx` 通过 `recordId` 从 Base 读取 `JSON数据` 渲染报告。
