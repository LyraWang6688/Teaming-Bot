## 根目录配置文件

| 文件 | 用途 | 说明 |
|------|------|------|
| `package.json` | 依赖管理 | 定义项目依赖和脚本 |
| `pnpm-lock.yaml` | 依赖锁定 | 锁定依赖版本 |
| `tsconfig.json` | TypeScript 配置 | TypeScript 编译器配置 |
| `next.config.ts` | Next.js 配置 | Next.js 运行配置 |
| `eslint.config.mjs` | ESLint 配置 | 代码规范检查配置 |
| `Dockerfile` | 容器构建 | 固化 Node/pnpm/Next.js 运行环境，便于迁移 |
| `docker-compose.yml` | 容器部署 | 将宿主机 `127.0.0.1:3011` 映射到容器 `3000` |
| `.env.production.example` | 环境变量样例 | 飞书、豆包 API、项目公网域名配置 |
| `drizzle.config.ts` | 数据库配置 | Drizzle 针对 Supabase Postgres 的 schema/generate/push 配置 |

## 核心链路

1. 手动入口：用户上传会议纪要，调用 `src/app/api/analyze/route.ts`。
2. 配置入口：`src/app/feishu-config/page.tsx` 展示飞书接入五步向导和初始化目标状态。
3. 用户级配置入口：`src/app/api/feishu/integrations/*` 负责按登录用户读写飞书集成配置。
4. OAuth 入口：`src/app/api/feishu/oauth/start/route.ts` 生成带 `state` 的授权链接；`src/app/api/feishu/oauth/callback/route.ts` 支持将 OAuth 结果直接写入数据库，同时保留旧的 env 回退模式。
5. 初始化入口：`src/app/api/feishu/integrations/[integrationId]/base/initialize/route.ts` 负责自动创建 Base、补齐会议信息表字段并回写当前集成。
6. 检查入口：`src/app/api/feishu/integrations/[integrationId]/checks/route.ts` 执行应用凭证、OAuth、Webhook、Base 联通的真实检查，并刷新状态面板。
7. 自动入口：飞书会议事件通过 `src/app/api/feishu/webhook/route.ts` 进入系统。
8. `analysisService.ts` 统一调用豆包 API，返回结构化分析结果。
9. `webhookProcessor.ts` 通过 `meeting_id -> recording -> minute_token -> transcript` 拉取逐字稿、调用分析服务，并回写多维表格。
10. `src/instrumentation.ts` 在服务启动时触发飞书链路恢复扫描。
11. `src/lib/feishu/monitor.ts` 统一输出飞书录制文件、导出、分析和恢复的结构化日志。
12. 报告页 `src/app/report/page.tsx` 通过 `recordId` 从 Base 读取 `JSON数据` 渲染报告。
